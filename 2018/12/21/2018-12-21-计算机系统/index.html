第1章 计算机系统漫游

最近开始学习计算机系统，目的是为了通过理解计算系统的运作原理，来写出更高效的代码程序。

1.1 什么是信息（bits+context）

像hello.c，由对应的ASCll码表示的字符文件，称为文本文件(context)。

其他所有文件都是二进制文件(bits)。

1.2 程序翻译执行的过程

翻译过程分为四阶段：预处理器、编译器、汇编器、链接器

  1. 预处理阶段：根据以字符#开头的命令，修改原始c程序
    e.g. #include <stdio.h>

  2. 编译阶段
    编译器（ccl）将文本文件hello.i翻译成文本文件hello.s, 包含一个汇编语言程序。

  3. 汇编阶段
    汇编器（as）将hello.s 翻译成机器语言指令，指令打包成relocatable object program 保存在hello.o文件中。

  4. 链接阶段
    调用标准C库中的一个函数（例如printf），由链接器（Id）合并到hello.o中，得到可执行文件hello文件。

1.5 高速缓存（cache memory）

高速缓存用来存储近期可能会需要的信息，它比寄存器大，比主存处理快，相当于寄存器和主存之间的桥梁，把近期经常用到的信息（信息量不算大的数据和指令）尽可能的在高速缓存中处理完成，提高系统的整体运行速度。

1.6 存储器层次结构

在寄存器和主存之间插入三层高速缓存，形成多级存储器层次结构，上一层的存储器是下一层的高速缓存（第6章）。

1.7 操作系统管理硬件

  1. 进程:操作系统对一个正在运行的程序的一种抽象

  2. 线程：组成进程的执行单元，在进程的contextz中，共享同样的代码和全局数据。

  3. 虚拟内存：虚拟内存为每个进程提供了一种假象，即每个进程都在独占的使用内存，实际每个进程的内存都是一样的，就如同杂技演员抛接球，每个进程就如同一个小球，球有四个，但每次在杂技演员手上只会有两个球，进程的只是交替的使用内存。

  4. 文件：文件就是字节序列

1.9 三个重要主题

Amdahl 定律

主要观点：要想显著加速整个系统，必须提升全系统中相当大的部分的速度

并发和并行

并行（parallelism）: 用并发使系统运行的更快

1）单核

2）多核，超线程：同时多线程
