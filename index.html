第2章 信息的表示与处理

第2章 信息表示与处理

无符号(unsigned)

补码(two’s compliments)

浮点数(floating point)

2.1 信息的存储

最小可寻址的内存单位： byte

虚拟内存：一串非常大的字节数组

虚拟地址空间：所有地址的集合

####16进制表示
16进制表示法：0x(0~F), 既解决了二进制表示法的冗长问题，也解决了十进制表示法与位模式转化困难的问题

字长(word size): 虚拟地址空间的最大大小

虚拟地址空间的由字长大小限制

32位字长：(2^32)4千兆字节

64位字长：(2^64)1.84*10^19

字数据大小

C 语言不同数据类型分配的字节数。

程序员应该力图使他们的程序在不同的机器和编译器上可移植。

寻址和字节序列

连续的地址序列：0x100, 0x101, 0x102, 0x103….

int a = 0x12345678

大端模式：最高有效字节在地址序列最前面，用于大多数intel兼容机

12 34 56 78

小端模式：最低有效字节在地址序列最后面，用于Android, Apple系统

78 56 34 12

bool 代数

& : and

| : or

~ : not

^ : exclusive or

加法逆元： x ^ x = 0，加法逆元是其本身

位向量：可以用来表示有限集合

e.g. a = [01101001] 可以用来表示有限集合 A =[0 3 5 6];

位级运算

  1. 交换运算：

    int inplace_swap(int *x, int *y){

    *y = *x ^ *y;

    *x = *x ^ *y;

    *y = *x ^ *y;

    }

  2. 掩码的应用

    掩码：0xFF

    0x89ABCDEF & 0xFF = 0x000000EF

  3. 移位操作
    左移，末尾补0

    右移：

  1. 逻辑右移：左端补k个零，用于无符号数
  2. 算术右移：左端补最高有效位的值1，对于有符号整数数据的运算特别有用

移位有限制，对于32位机器，移动k位, k <= 32

如果k 很大，左移 k mod 32 位，例如左移 40 位，就是左移8位

移位运算操作符的优先级问题

加法的优先级要大于移位

1 << 2+3 >> 4 相当于 1 << (2+3) >> 4;所以要养成加括号的习惯：(1<<2) + (3>>4)

###2.2 整数的表示

对于w位的机器

无符号整数：Unsigned

范围：0 ~ 2^w - 1

补码：Two’s complement

Tmax = 2^(w-1)-1

Tmin = -2^(w-1)

|Tmin| = |Tmax| + 1

有符号数和无符号数之间的转换：

位模式保持不变，但表示出的十进制数值有可能改变

在一个运算式中，如果有一个运算数是无符号的，另一个是有符号的，有符号数的会被隐性的转换成无符号数，造成非直观的结果。

-1 < 0u; 在32位机器中，-1被的位模式被解读成无符号数，2^32-1 < 0u, 得到的结果就是 0.

散列表(hash table)

算法图解: 散列表（Hash Table）

一种基本数据结构，它本质是一个数组，数组中的每个元素是一组键值对{key : value}，储存了键名称和值得信息，在python 中表示方式为 字典 （dictionary）。例如，dict1 = {key1: value1, key2: value2, key3: value 3…..}，输入键名称，通过散列函数确定索引位置，找到对应值

散列表的应用有电话簿查找，商品信息查找，缓存页面，投票防止重复等。

这种数据结构在元素几乎不发生冲突，均匀分布的时候查找时间是常数 O(1), 相对于简单查找的线性时间O(n) 和 二分查找的对数时间O(logn)要快不少。

如果散列表元素发生冲突，冲突的元素会以链表的形式占据在数组中的同一个位置，如果链表很长，会大幅降低散列表的查找速度。所以为了尽可能减少元素冲突，需要做到如下两点：

  • 降低填装因子
  • 选择良好的散列函数

所谓填装因子，是 散列表包含的元素数 / 位置总数， 一般经验是，如果填装因子大于0.7时，就resize数组大小。填装因子数越低，元素发生冲突可能就越低，分布越均匀。

散列函数决定了你不同的元素的最后的索引，不同函数会产生不同的索引分布。要根据输入元素的性质，选择合适的散列函数。

![](/assets/img/8 puzzle solving.png)

例如电池尺寸到功率的映射，其中电池尺寸为A、AA、AAA和AAAA，这个就可以选择以字符串的长度作为索引；

还有比如书名到作者的映射，其中书名分别为Maus、Fun Home和Watchmen，因为首字母不同，就可以字符串第一个字符为索引。

第1章 计算机系统漫游

第1章 计算机系统漫游

最近开始学习计算机系统，目的是为了通过理解计算系统的运作原理，来写出更高效的代码程序。

1.1 什么是信息（bits+context）

像hello.c，由对应的ASCll码表示的字符文件，称为文本文件(context)。

其他所有文件都是二进制文件(bits)。

1.2 程序翻译执行的过程

翻译过程分为四阶段：预处理器、编译器、汇编器、链接器

  1. 预处理阶段：根据以字符#开头的命令，修改原始c程序
    e.g. #include <stdio.h>

  2. 编译阶段
    编译器（ccl）将文本文件hello.i翻译成文本文件hello.s, 包含一个汇编语言程序。

  3. 汇编阶段
    汇编器（as）将hello.s 翻译成机器语言指令，指令打包成relocatable object program 保存在hello.o文件中。

  4. 链接阶段
    调用标准C库中的一个函数（例如printf），由链接器（Id）合并到hello.o中，得到可执行文件hello文件。

1.3 理解编译系统工作原理

理解链接时出现的错误（第7章）
避免安全漏洞（第3章）

1.4 shell 读取并储存内存中的指令

shell 是一个命令行解释器，可以读取hello程序的可执行文件。

e.g. Unix系统上运行该可执行文件

linux> ./hello #输入可执行文件程序位置

hello, world #读取程序并输出结果

linux> #程序终止，等待下一个命令行的输入

  4. 处理器：寄存器(Register)、程序计数器（PC）、算数/逻辑单元（ALU）

1.5 高速缓存（cache memory）

高速缓存用来存储近期可能会需要的信息，它比寄存器大，比主存处理快，相当于寄存器和主存之间的桥梁，把近期经常用到的信息（信息量不算大的数据和指令）尽可能的在高速缓存中处理完成，提高系统的整体运行速度。

1.6 存储器层次结构

在寄存器和主存之间插入三层高速缓存，形成多级存储器层次结构，上一层的存储器是下一层的高速缓存（第6章）。

1.7 操作系统管理硬件

  1. 进程:操作系统对一个正在运行的程序的一种抽象

  2. 线程：组成进程的执行单元，在进程的contextz中，共享同样的代码和全局数据。

  3. 虚拟内存：虚拟内存为每个进程提供了一种假象，即每个进程都在独占的使用内存，实际每个进程的内存都是一样的，就如同杂技演员抛接球，每个进程就如同一个小球，球有四个，但每次在杂技演员手上只会有两个球，进程的只是交替的使用内存。

  4. 文件：文件就是字节序列
